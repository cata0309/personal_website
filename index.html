Code

Jarvs

Jarvs is my biggest pet project yet. I first created jarvs to serve as an intuitive and quick ui for calling a couple scripts that I wrote to help manage the rvs program (more on that below). The idea grew into a very chipper personal assistant that just runs in the terminal. Jarvs is still in alpha, so there are many more functions to build. So far, Jarvs mostly utilizes pre-existing utilities.

You can read up on Jarvs and even download your own at my github repository.


RVS's

Research Visit Summaries (RVS's) are an important part of big research, especially in a clinical setting. I wrote two major scripts to help manage the rvs program I worked on. Jarvs ended up absorbing these functionalities.

You can find out more about RVS's and what attendings are at my other github repository.


MRI Blinder

Researchers often find themselves knowing too much about their own projects. I ran into this issue when I had to subjectively grade mri's and didn't want to impart any bias from knowing who belonged to the "test" group and who belonged to the "control" group. Of course this could have been accomplished by hand from someone external, but it would take hours to get through the number of files I was dealing with, even at a small-scale first-pass. Of course I wanted this to be very user friendly so that other researchers at the center could use it.

I have another github repository for this.
